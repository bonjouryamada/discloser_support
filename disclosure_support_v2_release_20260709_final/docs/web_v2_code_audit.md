## 指摘事項

### 中: EDINET/XBRLの財務数値抽出が誤値になり得る

対象: `web_v2/edinet_fetcher.py`

- `get_doc_id_for_date` は会社名が filerName に含まれる、または入力値が `secCode` に含まれるだけで一致扱いにしている（行44-48）。4桁証券コード入力時はEDINETの5桁コードに対する包含判定で広く拾える一方、意図しない提出者を拾う可能性がある。
- XBRL抽出は `currentyear` を含む全コンテキストから最初の値を採用し、連結タグを後から優先するだけで、期間/時点、連結/非連結、Abstract要素の除外、タグ粒度の選別が弱い（行122-178）。
- `netassetsabstract` も `net_assets` 候補に入っている（行169）。Abstract要素は通常数値ではないが、タグ判定としては危険。
- `unit_ref` と `decimals` を読み取っているが判定に使っていない（行139-154）。単位・scaleの誤解釈があると判定ラインが桁違いになる。
- 見つからない財務値を0として返す（行182-188）。UI上は0円基準で計算され、「取得失敗」と「実数0」を区別できない。

影響: 動的判定ラインが誤表示され、利用者が開示要否判断を誤る可能性がある。公開前に企業コード完全一致、対象タグホワイトリスト、コンテキスト選別、欠損値表示を強化するべき。

### 中: 数値ライン抽出の対象漏れが大きい

対象: `web_v2/disclosure_calculator.py`

- 割合計算の対象は限定的な語句だけ（行36）。「債務の総額」「連結会社の売上高」「発行済株式数」「1株当たり」「0.8/1.2」「100分の2.5」など、データ内に多数ある定量基準を拾えない。
- 固定金額は「億円」のみ（行46）。「千万円」「円以上/未満以外の表現」「50名」「25名」などは要約対象外。
- 要約は句点で分割するだけ（行63-69）なので、長文条文では文脈が過大または不足する。

影響: アプリ上の「数値ライン n件」は網羅性を示すものではない。表示文言に「自動抽出された一部」等の注記がないと、利用者が網羅済みと誤認しやすい。

### 中: 検索0件時に全件へ戻るため誤選択しやすい

対象: `web_v2/app.py`

検索・区分・manual条件で0件の場合、警告後に `filtered = records` へ戻して全101件を選択可能にしている（行81-83）。利用者は「条件に一致しない」と警告されつつ、直後のセレクトボックスで無関係な項目を選べる。

影響: 条件不一致時に誤った開示項目でレポートを作成するリスクがある。0件時は選択UIを止めるか、条件リセット操作を明示するべき。

### 中: テスト実行が現環境で確認できない

対象: `web_v2/tests`

既存テストは妥当なスモークテストを含むが、今回環境では Python ランタイム起動に失敗したため、テスト通過を確認できていない。公開前にCIまたは別端末で `python -m unittest discover -s tests -v` の通過を確認する必要がある。

### 軽微: HTML表示は概ねエスケープ済みだが運用監視が必要

対象: `web_v2/app.py`

`unsafe_allow_html=True` は複数箇所で使われている（行98-105、113-123、192）。データ由来の本文は `criteria_html` で `html.escape` 後に `<strong>` と `<br>` のみ再付与しており（行65-69）、主要なXSS対策は入っている。イベントカードも選択データを `html.escape` している（行116-119）。

ただし、今後JSONデータを外部入力化する場合は、`unsafe_allow_html=True` の使用箇所を固定HTML/エスケープ済み本文に限定するルールを維持すること。

### 軽微: APIキー混入は未検出

対象: `web_v2`, `data`, `docs`

`EDINET_API_KEY` の参照は README/DEPLOYMENT と実装上の環境変数・Streamlit secrets 読み込みのみ。平文キーらしき値は検出していない。README/DEPLOYMENTにも `.env` やAPIキーをコミットしない注意がある。

### 軽微: 入力パース失敗が0扱いになる

対象: `web_v2/app.py`

財務入力 `parse_number` は変換失敗時に0.0を返す（行58-62）。誤入力が「未入力/0」と同じ扱いになり、判定ラインが0百万円で表示され得る。入力エラーとして明示した方が安全。

### 軽微: 未使用変数・例外握りつぶし

対象: `web_v2/data_service.py`, `web_v2/edinet_fetcher.py`

- `data_service.py` の `normalized_key` は代入されるが使われていない（行28）。
- `edinet_fetcher.py` は Streamlit secrets 読み込み失敗やXBRL値変換失敗を `pass` で握りつぶす箇所がある（行20-23、179-180）。本番障害調査時に原因が追いにくい。

### 軽微: Excel出力のテストが表示内容の妥当性まで見ていない

対象: `web_v2/report_export.py`, `web_v2/tests/test_report_export.py`

Excel生成は開けることのみ検証されている。手動レビュー時の色付け、財務数値、注意文言、動的判定ライン、文字化けや長文折り返しは未検証。
